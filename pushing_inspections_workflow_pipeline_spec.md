## Core Principle

`PushingInspections` is the evidence lane.

Its job is to convert uncertainty into condition truth:

- what the asset is
- what is wrong
- what is needed
- what can wait
- what the next downstream lane should be

This lane feeds:

- parts
- repair
- transport
- title/DMV readiness
- finance or insurance decisions when condition affects value or risk

## Best First Workflow Shape

Workflow name:
- `pushing_inspections_evidence_orchestration_v1`

Workflow intent:
- take an asset from raw concern to verified findings, then route cleanly into repair, parts, transport, or closeout

Pipeline stack:

1. `p242835887_service_requests`
   - raw concern and intake shell
2. `p242835887_vehicle_inspections`
   - evidence and findings shell
3. `Auto Repair` on `tickets`
   - execution controller when the inspection produces operational work
4. generic task pipeline
   - photos, callbacks, shop chase, missing docs, retest

## Stage Gates

### Before leaving `Intake`

All must be true:
- raw problem is stated clearly enough to classify
- asset identity is usable
- inspection shell exists

### Before leaving `Inspection`

All must be true:
- evidence exists
- findings are itemized enough to make a decision
- the next downstream route is explicit

### Before leaving `Repair Handoff`

All must be true:
- repair, parts, transport, or closeout path is chosen
- responsible execution record exists if work continues

### Before leaving `Closeout`

All must be true:
- client received the outcome
- operational memory is written back

## Related Doctrine To Keep

`PushingStandby` already gives the right mental model:

- `PushingInspections` = diagnostics / telemetry / evidence
- `PushingSubcontractors` = field execution network

## Big-Picture Rule

Inspections should not be modeled as "just a body shop task."

They are the evidence engine for the automotive control plane.

The platform wins when every inspection can answer:

- what is the asset
- what is wrong
- what is proven
- what must happen next
- which lane now owns the work
